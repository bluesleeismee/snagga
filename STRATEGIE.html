Stand: Juni 2026 · Intern · Nicht veröffentlichen

snagga.de ist keine Preis-Massenplattform, sondern eine kuratierte Deal-Seite mit Qualitätsanspruch. Ziel: echte Preisvorteile sichtbar machen, Vertrauen aufbauen, hohe Klickwahrscheinlichkeit erzeugen.

prüfen → bewerten → ranken → kuratieren → anzeigen

FBA-Proxy

Formeln

// Abstand 90-Tage-Ø
faktor_avg = max(0, (avg90 - current) / avg90)

// Abstand ATL
faktor_atl = max(0, (current - atl) / avg90)   // niedriger = besser

// Popularität — Sales Rank INVERTIERT und NORMIERT
rank_faktor    = 1 - (sales_rank / max_kategorie_schwellwert)
rating_faktor  = (rating - 4.0) / 1.0
review_faktor  = log10(reviews) / log10(10000)
popularitaet   = rank_faktor × 0.5 + rating_faktor × 0.3 + review_faktor × 0.2

// Stabilität
stabilitaet = 1.0   // Preis seit ≥ 24h stabil
stabilitaet = 0.3   // Preis < 24h auf diesem Niveau (Ausreisser)

// Gesamt-Score
score = (faktor_avg × 0.40 + faktor_atl × 0.30 + popularitaet × 0.20 + stabilitaet × 0.10) × 100

Score < 40 → Deal wird nicht angezeigt.

08 Update-Logik

1h

Stündlich — Keepa /deals + Live-Check

Keepa /deals abrufen → Hard Filters → Score berechnen → aktive Deals via Amazon-Seite preisprüfen → Score < 40 deaktivieren → Backup verifizieren und nachrücken → Top 10 neu setzen → Tags neu setzen

30m

Alle 30 Minuten — Top Picks Preis-Check

Nur die 10 Top Picks werden häufiger geprüft als reguläre Deals.

03h

Nächtlich 03:00 Uhr — Deep-Sync via Keepa /product

Alle 300 Deals (aktiv + Backup) vollständig aktualisiert: Sales Rank, 90-Tage-Ø, 180-Tage-Ø, ATL, Rating, Reviews, Preishistorie. Scores und Tags neu berechnet.

Backup-Verifikation (Race Condition verhindert)

// Bevor ein Backup-Deal live geht:
1. Live-Preis prüfen (Amazon-Seite)
2. Preis noch gültig? → is_active = true
3. Preis nicht mehr gültig? → nächstes Backup prüfen
