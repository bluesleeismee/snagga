🏗️1. System-Übersicht

Snagga ist eine Progressive Web App (PWA) zur Anzeige historisch günstiger Amazon-Deals für den D-A-CH-Raum. Nutzer können Deals nach Kategorie filtern, sortieren, die Preishistorie einsehen und per Affiliate-Link kaufen.

🚀9. Deployment-Workflow

Normaler Code-Push

# Änderungen lokal testen
cd D:\Claude\DealRadar\frontend
npm run dev

# Commit & Push → Auto-Deploy
git add .
git commit -m "feat: Beschreibung"
git push

Nach dem Push deployt Vercel das Frontend automatisch (~30 Sek.) und Render das Backend (~2 Min.).

Commit-Konventionen

PrefixBedeutungBeispiel
feat:Neue Funktionfeat: Kategorie-Filter hinzugefügt
fix:Bugfixfix: Swipe-Back schliesst Modal korrekt
style:UI/CSS-Änderungstyle: Sidebar Farben angepasst
db:Datenbank-Schemadb: Keepa-Felder hinzugefügt
ops:Infrastrukturops: robots.txt noindex gesetzt

Frontend lokal starten

cd D:\Claude\DealRadar\frontend
npm install          # einmalig
npm run dev          # http://localhost:5173

Backend lokal starten

cd D:\Claude\DealRadar\backend
# .env mit DATABASE_URL anlegen
pip install -r requirements.txt
uvicorn main:app --reload  # http://localhost:8000

🔑
Keepa Plan €49/Mo
ca. 1.800 Tokens/Tag

1 Token = 1 vollständiges Produkt inkl. 2 Jahre Preishistorie. Tokens verfallen nach 1 Stunde. Batch: max. 100 ASINs.

⏱️
Update-Strategie
Prioritätsbasiert

Prio 1 (stündlich): Hot Deals <50€
Prio 2 (6h): Normale Deals
Prio 3 (täglich): Slow Movers

Geplante Felder in products

Die Supabase-Tabelle hat bereits alle nötigen Spalten: price_history_keepa (JSONB), keepa_last_fetched, update_priority, highest_ever, discount_pct, sales_rank.

Geplante Library

# AsyncKeepa für FastAPI
pip install keepa

# Beispiel: Batch-Abruf
import keepa
api = keepa.AsyncKeepa('API_KEY')
products = await api.query(['ASIN1', 'ASIN2'], history=True)
